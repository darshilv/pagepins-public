Automate with PagePins Companion Bridge

Keep the Chrome extension flow exactly as-is, then optionally add a local automation layer that routes annotations into Codex/Cursor/Claude workflows.

View Companion Bridge on GitHub Clone URL

HTTP Bridge Service

The bridge receives extension snapshots at POST /pinpoint, stores item-level records, and tracks lifecycle changes.

Runtime: src/server.js

MCP Server

A local MCP stdio server exposes annotation tools for assistants while reading from the same SQLite store.

Runtime: src/mcp-server.js

Skills + Client Rules

Includes Codex skill packs and setup scripts so teams can quickly route requests through pagepins_assist.

Install: npm run setup:mcp

Automation Flow

1. Capture

Create, edit, resolve, and delete notes directly in PagePins on live UI.

2. Sync

Companion Bridge ingests snapshots and manages workflow states: pending, reviewed, sent.

3. Execute

Assistants pull pending items, draft implementation prompts, and update status as work progresses.

Quick Start (from README)

git clone https://github.com/darshilv/pagepins-companion-bridge.git
cd pagepins-companion-bridge
cp .env.example .env
npm install
npm run dev

Bridge URL: http://127.0.0.1:3456

Second Terminal + Setup

npm run mcp
npm run setup:mcp

In extension settings, enable Companion Bridge and set endpoint:

http://127.0.0.1:3456/pinpoint

Verify Ingestion

curl http://127.0.0.1:3456/health
curl "http://127.0.0.1:3456/annotations?limit=20"
curl "http://127.0.0.1:3456/annotations/current"

Assistant Actions Included

  • inbox for pending annotation triage
  • open and focus for item context
  • draft for implementation-ready chat output
  • set_status for workflow transitions

Privacy and Control

This automation layer is local-first. Extension note data stays in browser storage, and bridge records live in your local runtime. You choose when the bridge runs and which assistants can access tools.
